Methodology: RAG accuracy is assessed via embedding coverage (percentage of chunks with valid vector representations), topic coverage in the textbook corpus, and the distribution of chunks available for semantic retrieval. Cosine similarity is computed using L2-normalized embeddings (all-MiniLM-L6-v2, 384-dim) via numpy np.dot.

Methodology: Model accuracy is measured by pipeline success rates (summary, quiz, recommendations), question topic coverage, and provider reliability. The system uses MiniMax-M2.7 as primary with Gemini-2.5-flash-lite as fallback. Success = a non-empty, structurally valid output was produced and persisted. Question quality is proxied by topic diversity and user performance.

User Performance & Question Quality Proxy

Methodology: Since ground-truth labels for AI-generated questions are unavailable, we use user performance as a proxy for question quality. Well-calibrated questions should produce a range of scores centered around 50-70%. Topic-wise accuracy below 40% may indicate questions are misaligned with the source material or too difficult.

Data Transparency Notice

All metrics on this page are computed in real-time from the live database. No cached or synthetic values are used. The evaluation methodology follows standard information-retrieval and machine-learning evaluation practices:

  • Success rates are calculated as successes / (successes + failures)
  • Embedding coverage is embedded_chunks / total_chunks
  • User performance is a proxy metric; no human annotation ground truth is available
  • GenerationMetric records are captured per AI provider call when instrumentation is enabled
